# My Essential Front-end Links
A curated list by <a href="https://twitter.com/aarongarciah/" target="_blank">Aarón García</a>. Suggest a link with a PR or open a [new issue](https://github.com/aarongarciah/essential-frontend-links/issues/new).


## HTML


### Reference

**[Can I Use](http://caniuse.com/)**  
"Can I use" provides up-to-date browser support tables for support of front-end web technologies on desktop and mobile web browsers.

**[Codrops CSS Reference](http://tympanus.net/codrops/css_reference/)**  
An extensive CSS reference with all the important properties and info to learn CSS from the basics

**[SitePoint Sass Reference](http://www.sitepoint.com/sass-reference/)**  
An attempt at making Sass more approachable by providing short yet insightful entries for all buzzwords.


### Generators

**[Box Shadow Generator](http://www.cssmatic.com/box-shadow)**


### Sass Mixins & Libraries

**[Breakpoint](http://breakpoint-sass.com/)** [:octocat:](https://github.com/at-import/breakpoint)  
Breakpoint makes writing media queries in Sass super simple.

**[Sass MQ](http://sass-mq.github.io/sass-mq/)** [:octocat:](https://github.com/sass-mq/sass-mq)  
A Sass mixin that helps you compose media queries in an elegant way.


### Sass Docs
**[SassDoc](http://sassdoc.com/)**  
SassDoc is to Sass what JSDoc is to JavaScript: a documentation system to build pretty and powerful docs in the blink of an eye.


### Styleguides/Guidelines

**[CSS Guidelines](http://cssguidelin.es/)** by [@csswizardry](https://twitter.com/csswizardry "Harry Roberts")  
High-level advice and guidelines for writing sane, manageable, scalable CSS

**[Sass Guidelines](http://sass-guidelin.es/)** by [@hugogiraudel](https://twitter.com/HugoGiraudel "Hugo Giraudel")  
An opinionated styleguide for writing sane, maintainable and scalable Sass.

**[Code Guide](http://codeguide.co/)** by [@mdo](https://twitter.com/mdo "Mark Otto")  
Standards for developing flexible, durable, and sustainable HTML and CSS.

**[Idiomatic CSS](https://github.com/necolas/idiomatic-css)**  by [@necolas](https://twitter.com/necolas "Nicolas Gallagher")  
Principles of writing consistent, idiomatic CSS.

**[Primer](http://primercss.io/guidelines/)**  
The CSS toolkit and guidelines that power GitHub.

**[Airbnb](https://github.com/airbnb/css)**  
A mostly reasonable approach to CSS and Sass.

**[Trello CSS Guide](https://gist.github.com/bobbygrace/9e961e8982f42eb91b80)**  

**[Google HTML/CSS Style Guide](https://google.github.io/styleguide/htmlcssguide.xml)**  

**[18F](https://pages.18f.gov/frontend/)**  
Here you will find all of the guides produced by 18F's (U.S. Federal Government) front end guild.
